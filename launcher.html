Play

Before playing: make sure Ollama is running, launch STS2 on Steam, accept the mods prompt, and stay on the main menu.

Single Game

Interactive TUI — plays one run, then stops. Great for watching the bot in action.

A = Champion B = Challenger
cd ~/AJS_CTS/ClawTheSpire && bash play.sh --profile a
cd ~/AJS_CTS/ClawTheSpire && bash play.sh --profile b

Add --step to step through each action

Batch (Continuous)

Runs games back-to-back automatically. Ideal for gathering data overnight.

A = Champion B = Challenger
cd ~/AJS_CTS/ClawTheSpire && bash play.sh batch --profile a
cd ~/AJS_CTS/ClawTheSpire && bash play.sh batch --profile b

Add --once for a single game via batch runner

Start the bot BEFORE beginning a run so it picks the character.

Press Ctrl+C in Terminal to stop the bot at any time.

Profile A is the known-good config. Edit config_b.py to experiment, then test with profile B.

Encounter Report

Scan gameplay logs to find which enemies the bot has faced vs. what it's been trained on. Use this to identify training gaps.

Full Report

Shows all encounters: trained and untrained, with win rates and frequency.

cd ~/AJS_CTS/ClawTheSpire && python3 encounter-report.py

Training Gaps Only

Shows only encounters NOT in the training set — the highest-priority candidates for improvement.

cd ~/AJS_CTS/ClawTheSpire && python3 encounter-report.py --untrained

Run after a play session to see new enemies. More games = more data = better insights.

Code

Use Claude Code in a Docker sandbox to safely modify the bot. Changes sync to your Mac and are tracked in git.

  1. Make sure Docker Desktop is running.
  2. Start and enter the sandbox:
    cd ~/AJS_CTS/ClawTheSpire && docker compose up -d && docker compose exec sts2 bash
  3. Run Claude Code:
    claude --dangerously-skip-permissions
  4. When done, type exit to leave. Changes sync automatically.

Save to GitHub

Commit and push your changes to your fork.

cd ~/AJS_CTS/ClawTheSpire && git add -A && git commit -m "describe changes" && git push origin andrew-dev

Pull Friend's Updates

Get the latest changes from your friend's repo.

cd ~/AJS_CTS/ClawTheSpire && git fetch upstream && git merge upstream/main

Config Profiles

A/B test different strategy configs. Edit the challenger, run games with each, compare results.

A — Champion

Baseline Config

The known-good settings. Don't edit this — it's your control group.

sts2-solver/src/sts2_solver/config_a.py
B — Challenger

Survive & Scale

+33% block weight, +40% Weak value, Leg Sweep/Noxious Fumes to S-tier, leaner deck target (10), more conservative resting. Hypothesis: surviving longer = winning more.

sts2-solver/src/sts2_solver/config_b.py

Promote Challenger

When B outperforms A, copy it over to become the new champion:

cd ~/AJS_CTS/ClawTheSpire/sts2-solver/src/sts2_solver && cp config_b.py config_a.py

Then reset config_b.py for the next experiment.
